# 考勤报表系统 使用文档

## 一、系统概述

考勤报表系统是一款 Web 应用，用于将钉钉导出的上下班打卡日报进行可视化展示，并生成标准格式的考勤统计报表。

**主要功能：**
- 导入钉钉打卡日报 xlsx 文件
- 分页查看概况统计和打卡详情数据
- 按部门导出考勤数据统计报表
- 按部门导出周内加班统计报表

---

## 二、启动系统

### 2.1 启动后端

```bash
cd backend
python main.py
```

后端服务启动后监听 `http://localhost:8000`，可访问 `http://localhost:8000/docs` 查看 API 文档。

### 2.2 启动前端

```bash
cd frontend
npm run dev
```

前端开发服务器启动后访问 `http://localhost:5173`。

> **注意：** 前后端需同时运行，系统才能正常工作。

## 三、使用流程

### 第一步：导入数据

1. 打开浏览器访问 `http://localhost:5173`，系统自动跳转到**数据导入**页面

2. 将钉钉导出的 `上下班打卡_日报_YYYYMMDD-YYYYMMDD.xlsx` 文件拖拽到上传区域，或点击上传区域选择文件

   ![上传页面示意]
   ```
   ┌────────────────────────────────────────────┐
   │                                            │
   │          ↑  拖拽 xlsx 文件至此处            │
   │             或 点击上传                     │
   │                                            │
   │    仅支持钉钉导出的上下班打卡日报 .xlsx 文件  │
   └────────────────────────────────────────────┘
   ```

3. 上传成功后页面显示解析结果摘要：

   | 字段 | 说明 |
   |------|------|
   | 总人数 | 本期打卡的员工总数 |
   | 日期范围 | 数据覆盖的时间段 |
   | 概况统计记录 | 概况统计 sheet 的有效数据行数 |
   | 打卡详情记录 | 打卡详情 sheet 的有效数据行数 |
   | 包含部门 | 数据中出现的所有部门 |

4. 点击 **"前往数据预览"** 或 **"前往报表导出"** 按钮继续操作

---

### 第二步：查看数据（可选）

点击左侧导航 **数据预览**，可查看已导入的原始数据。

#### 概况统计 Tab

展示每人每天的考勤概况，包含：时间、姓名、部门、班次、考勤结果、迟到/早退/旷工次数、加班状态、各类假期天数等。

**筛选操作：**

| 筛选项 | 说明 | 示例 |
|--------|------|------|
| 搜索姓名 | 模糊匹配姓名 | 输入 `张` 显示所有姓张的员工 |
| 部门筛选 | 模糊匹配部门名称 | 输入 `示例` 筛选示例部门 |

输入后点击 **查询** 按钮，或清空后点击查询显示全部。

每页显示 50 条，底部分页栏可翻页。

#### 打卡详情 Tab

展示每次打卡的详细记录，包含：日期、姓名、部门、打卡类型（上班/下班）、应打卡时间、实际打卡时间、打卡状态、打卡地点等。

**筛选操作：**

| 筛选项 | 说明 | 示例 |
|--------|------|------|
| 搜索姓名 | 模糊匹配姓名 | `员工B` |
| 部门筛选 | 模糊匹配部门 | `研发中心` |
| 日期 | 模糊匹配日期字符串 | `2026/01/15` |

---

### 第三步：导出报表

点击左侧导航 **报表导出**，页面提供两种报表的导出。

#### 3.1 考勤数据统计报表

生成包含两张工作表（Sheet）的 Excel 文件，底层使用模版生成：
- **Sheet 1（明细表）**：每人占两行（上午/下午），包含 31 天每天上下午的出勤符号及基础月度汇总。
- **Sheet 2（统计表）**：包含针对本次查询部门所涉及的全量员工名单的请假、迟到、矿工等各项出勤异常类别的按人次度量汇总。

**符号说明：**

| 符号 | 含义 |
|------|------|
| √ | 正常出勤 |
| ● | 迟到 |
| ◎ | 早退 |
| × | 旷工 |
| ○ | 缺卡 |
| ☆ | 出差 |
| △ | 事假 |
| ◇ | 病假 |
| ※ | 年假 |
| ＃ | 调休假 |
| Ω | 婚假 |
| ▽ | 产假 |
| ÷ | 丧假 |
| （空） | 休息日 |

**操作步骤：**
1. 在 **部门** 输入框填写部门关键词（如 `示例部门`），或留空导出全部人员
2. 点击 **导出 xlsx** 按钮
3. 浏览器自动下载文件，文件名将根据上传数据自动提取真实年月，格式如：`{部门}_{年}年{月}月考勤数据统计.xlsx`

#### 3.2 周内加班统计报表

生成同样包含两个工作表的 Excel 文件：
- **Sheet 1（明细表）**：每人占两行，包含每天 20:00-22:00 区间的实际下班时间及 22:00 之后的实际下班时间。
- **Sheet 2（统计表）**：针对全量员工名册，统计并输出每人在这两段加班区间内的月度总加班频次。

> **统计规则：** 仅统计实际下班打卡时间在 20:00 之后的记录。休息日和节假日不在此统计范围内（取决于原始打卡数据）。

**操作步骤：**
1. 在 **部门** 输入框填写部门关键词，或留空导出全部
2. 点击 **导出 xlsx** 按钮
3. 浏览器自动下载，文件名将自动提取真实年月，格式如：`{部门}_{年}年{月}月周内加班统计.xlsx`

## 四、数据源要求

系统接受钉钉考勤管理后台导出的 **上下班打卡_日报** xlsx 文件，该文件须包含以下两个 Sheet：

| Sheet 名 | 说明 |
|----------|------|
| 概况统计与打卡明细 | 每人每天一条记录，含考勤结果、异常统计、加班统计、假勤统计 |
| 打卡详情 | 每次打卡一条记录，含实际打卡时间和打卡状态 |

> **注意：** 每次导入新文件会覆盖上一次的数据，重启后端服务后需重新上传。

## 五、考勤符号规则配置

如需修改考勤符号的映射规则（例如新增状态类型），编辑 `backend/config.json` 中的 `attendance_symbols` 字段：

```json
{
  "attendance_symbols": {
    "正常": "√",
    "迟到": "●",
    "早退": "◎",
    "旷工": "×",
    ...
  }
}
```

修改后重启后端服务即可生效，无需修改代码。

## 六、常见问题

**Q：上传后提示"解析失败"？**
A：确认上传的文件是钉钉导出的原始格式，不要在 Excel 中另存为或修改格式。

**Q：数据预览显示"请先上传数据文件"？**
A：后端服务重启后内存缓存清空，需重新上传文件。

**Q：导出的报表人员顺序与模版不一致？**
A：系统优先按模版中原有的人员顺序排列，模版中没有的新人员追加在末尾。

**Q：部门筛选不生效？**
A：部门输入框支持模糊匹配，输入部门名称的一部分即可，如 `示例` 匹配 `示例科技有限公司/示例部门`。

**Q：22:00 之后加班次数为 0？**
A：检查该人员当月是否有 22:00 之后的下班打卡记录，且打卡状态正常。

## 八、版本更新记录

**v1.1.0 (2026-03-10)**
- **动态年月解析**：自动从上传打卡记录的有效日期中提取年份和月份，生成的 Excel 文件名及 Sheet 名称将根据实际数据动态命名（例：从固定 `2026年1月` 变更为实际打卡年月的动态名称）。
- **自动化统计汇总表（Sheet 2）**：在考勤与加班明细表的基础上，自动化抓取全量员工列表（包含本月无休/无加班记录员工），并在导出的第二张工作表（Sheet 2）中自动完成异常指标度量（迟到、早退、各类请假）和不同时段加班次数的汇总统筹。
- **去掉硬编码文案与文件名修复**：清除了原本写死在考勤和加班模板中的部门名（示例部门），同时修复了 Vue 客户端导出功能对月份的强制重写，使其正确获取基于 `Content-Disposition` 传送的实际名称。
